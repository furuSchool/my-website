## 基本
:::tip[伝送路符号化]
伝送路符号化とは，信号を物理的な伝送路（ケーブルや無線など）に適応させるための符号化する技術。通信路符号化とは異なる。
- 通信路符号化：誤り訂正などで伝送路上で発生する誤りを検出・訂正する技術
:::

デジタル信号をどうやって波形に乗せるか。
- チャネル（伝送路）は周波数に制限があり，ビット同士が影響しないようにする必要がある（ISI：符号間干渉）

#### 考えるべきこと
- 同期がとりやすい？
- 雑音に強いか？
- 直流成分があるか？（変成器に弱い）
- 誤り訂正があるか？
- 伝送効率は？
- 信号の帯域幅は？
- 電力は？ $\rightarrow$ パワースペクトル密度（PSD）を考える

#### 特徴
- RZ：クロックが取りやすい（同期がとりやすい），polar(RZ)なら絶対値取れば Timing Signal になる
- on-off：直流成分があり電力効率が悪い
- Alternate Mark Inversion (Bipolar) Signaling（バイポーラ符号）：誤り検出がしやすい。直流成分がない。帯域幅が狭い。電力が大きい
    - 初期の PCM で使われた
- Split Phase (Manchester) Encoding（マンチェスター符号）：直流成分がない。同期が取りやすい。帯域幅が広い
    - Ethernet で使われる

## ISI（符号間干渉）
：パルスは物理的な伝送路を通ると広がる・遅延するため，他のパルスに影響を与える $\rightarrow$ どうやって干渉を防ぐか？  
参考資料：https://en.wikipedia.org/wiki/Nyquist_ISI_criterion


#### サンプリング点で ISI を 0 にする
パルスは周波数帯域では無限に広がっている $\rightarrow$ 例えばナイキストパルス（sinc 関数の近似）を使う
- こうすれば，所望のサンプリング点以外で 0 になる

#### チャネルの帯域幅
時間領域に広いパルスを使うほど，周波数領域では狭くなる（フーリエ変換より）。しかし，その分時間方向の ISI が大きくなる。
$\rightarrow$ うまく調整して所望のサンプリング点以外で 0 にすれば良い。 

## アイダイアグラム
信号波形の遷移を一定時間間隔で多数サンプリングし，重ね合わせたもの。ISI や雑音の影響を見るのに使われる。

- 水平方向の目の空き（horizontal opening eye）：ジッタ（タイミングのずれ）や，どの瞬間ならサンプリングしてもいいかがわかる
